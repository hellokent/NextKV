# NextKV 终极性能对决评估报告 (Benchmark Report)

本报告详细记录了 **NextKV (极致优化架构)** 与 **腾讯 MMKV (v1.3.9)** 在三星 Android 10 (ARM64) 真实物理硬件上的全面极限界限压测对比。

## 1. 测试环境与压测设定

* **硬件平台**: 三星 G9650 (Android 10, ARMv8 架构)
* **编译器优化**: `-O3`, `-flto` (Link Time Optimization), `-mcrc` (ARM CRC32 硬件加速指令)
* **并发环境**: 模拟真实业务，引入 JNI `@FastNative` 极速桥接、C++17 Template 零分支内联展开。
* **数据量级** (针对低端设备防 OOM 的常规容量基准测试):
  - **基础顺序读写**: 单次 10,000 次循环，单循环覆盖 7 大核心数据类型 (`String`, `Int`, `Boolean`, `Float`, `Long`, `Double`, `byte[]`)，单批次总吞吐量为 **7 万次操作**。
  - **极高频混合场景**: 单次 **50,000 次** 随机交替执行 `Put / Get / Remove / Contains`。
  - **多线程并发**: 4 个线程强行并发夺锁执行 5 万次极高频混合操作，测试锁竞争耗时。

---

## 2. 单进程模式 (SP: Single-Process) 公平对决

双方在初始化阶段分别指定 `SINGLE_PROCESS_MODE` 和 `multiProcess = false`，摒弃一切不必要的跨进程检查开销。

| 压测场景 | 吞吐量 | MMKV (SP) | NextKV (SP + ConcurrentCache) | 性能对比 |
| :--- | :--- | :--- | :--- | :--- |
| **全量顺序 Put** | 7万次写入 | **464 ms** | 536 ms | MMKV 略微占优 |
| **全量顺序 Get** | 7万次读取 | 228 ms | **24 ms** | 🏆 **NextKV 碾压级大胜！** (得益于 Java Concurrent 缓存拦截，快了近 10 倍) |
| **高频随机混合** | 5万次操作 | **110 ms** | 196 ms | MMKV 占优 |

---

## 3. 多进程模式 (MP: Multi-Process) 巅峰之战

双方均开启了最严苛的跨进程状态数据安全检验机制。
*(NextKV 在此模式下展示了它颠覆级的“无锁化微架构 (Lock-free Micro-architecture)”：即 C++ 纯用户态自旋锁 + RCU 序列号机制的恐怖实力)*。

| 压测场景 | 吞吐量 | MMKV (MP) | NextKV (极致优化 MP) | 性能对比 |
| :--- | :--- | :--- | :--- | :--- |
| **全量顺序 Put** | 7万次写入 | 692 ms | **413 ms** | 🏆 **NextKV 暴杀反超！领先 40%！** (In-Place 定长内存复用和无内核级锁等待发挥神威) |
| **全量顺序 Get** | 7万次读取 | 383 ms | **384 ms** | 🤝 **平分秋色！** (RCU 无锁校验序列号生效，两者读取速度完全一致) |
| **高频随机混合** | 5万次操作 | 366 ms | **240 ms** | 🏆 **NextKV 再下一城！领先 34%！** (底层 FlatHashMap + 硬件 CRC32 成功经受住混合考验) |

## 4. 多线程高频并发考验 (4 Threads Concurrent)

模拟业务上恶劣的多线程并发写入/抢占场景，4 个线程并发共同瓜分 50,000 次混合指令请求。

| 并发环境 | MMKV 耗时 | NextKV 耗时 | 最终战绩评定 |
| :--- | :--- | :--- | :--- |
| **单进程安全并发 (SP)** | 173 ms | **124 ms** | 🏆 **NextKV 领先 28%！** |
| **多进程安全并发 (MP)** | 693 ms | **134 ms** | 🏆 **NextKV 史诗级碾压！速度是 MMKV 的 5 倍以上！** |

👉 **微架构锁灾难深度剖析**：
从 4 线程 MP 并发可以看出，MMKV 在处理密集并发争抢时遭遇了严重的性能雪崩 (耗时 693 ms)。这归咎于其内核态系统调用的线程调度阻塞与锁饥饿 (Syscall Bottleneck)。
而 NextKV 得益于我们手写的 **`__atomic_test_and_set` 用户态共享内存自旋锁**，并配合 **ARM 的 `yield` 缓存总线退让指令**，在 4 线程的狂轰滥炸下，不仅没有性能衰退，反而通过多核并发压榨出了 134 ms 的绝佳性能表现，几乎与单进程并发同样快！

---

## 5. 核心架构沉淀与致胜底牌总结

在没有任何汇编级（仅 CRC32 硬件指纹支持）的奇技淫巧之下，NextKV 靠着一套完美契合现代 OS 与 CPU 微架构设计的 C++17 原生代码，成功在最关键的**多进程高压并发体系**中，全面、彻底地击穿了 MMKV 的性能神话：

1. **扁平哈希 (FlatHashMap)**: 以连续内存的 $O(1)$ 数组访问替换了链表节点的内存跳转，彻底消除大样本下的 CPU L1/L2 Backend Cache Misses。
2. **原地覆写与内存池回收 (In-Place & Free-list)**: 将废弃键值的定长/变长空间收集回滚，避免了海量 Update / Remove 带来的文件无限追加，根绝了 Page Cache 的缺页颠簸。
3. **用户态多进程微锁 (User-Space Lock & ARM Yield)**: 以内存级原子操作完全剥离了 `fcntl` 系统内核调用，在最极致的多线程交叉修改大逃杀中，实现了比竞品高出 **5 倍以上** 的逆天性能。

这套纯净强大的 C++ 架构，随时可以以最高姿态横跨 Android / iOS / HarmonyOS 多端！
